Developing Identifiers for Heritage Collections

This resource describes the different aspects of developing persistent identifiers (PIDs) in the context of Heritage Collections.

A persistent identifier is a long lasting reference to a resource, such as a collection item, book, or object. They can also refer to canonical metadata, e.g. a creator of an object.

Examples of PIDs include archival resource keys (ARKs), persistent URIs and digital object identifiers (DOIs). Generally a persistent identifier can be presented as an actionable link which can be used to access the resource or metadata about the resource.

Why use PIDs?

Persistent identifiers have a number of potential benefits. They allow you to reliably link to your collection around the web and in publications. They also make it easier to track the use of your resources where they are cited elsewhere.
